En este trabajo aplicamos un algoritmo iterativo que analiza
una gran cantidad de datos en el menor tiempo posible
Usaré el algoritmo k-medias para generar grupos y calcular:
    -datos
    -medias
    -minimo
    -maximo 
    -varianza
datos de entrada:
    UINT32 – Number of rows - 04 bytes
    UINT32 – Number of columns - 04 bytes
    foreach row:
        foreach column:
            REAL32 -Value - 04 bytes
        end
    end

Actualización de los cetroides
    -cada vez que el conjuto datos de un grupo cambie, hay que acrtualizar su centroide
    todos los nodos deben comunicarse las actualizaciones
Actualización de los grupos 

Con el genrador de puntos que se nos ha proporcionado se ha generado un lista de muestreo con lo que probar el programa

primera implementación (secuencial sin mejoras de rendimiento)
para esta implentacion he investigado el algoritmo k-medias y como funciona:
-primero elegimos k puntos iniciales y las utilizamos como centroides iniciales
-generamos k grupos asociandole el punto con la media mas cercana
-recalculamos el centroide de cada grupo 
-repetimos hasta que se logre la convergencia

con la implementación secuencial el progrma tarda al rededor de 3.29e-06 segundos 

la segunda implementación con optimización de metaprogramacion el progrma tarda 1.62e-06 segundos

Mejoras iniciales con optimización en seria y metaprogramacion:
    -Parametro en platilla de kGrupos lo que permite loop unrolling
    -Localidad de datos, centroides se guardan en el Stack maximizando el uso de la cache
    -Eliminación de operaciones costosas como pow y sqrt
    -Paso por referencia constante

la segunda implementación con optimización de metaprogramacion el progrma tarda 1.62e-06 segundos
lo cual implica una mejora del 50.8% aprximadamente
